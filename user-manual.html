1. INTRODUCTION

1.1 Purpose of the Manual

This Software User Manual (SUM) is the official reference guide for Hanapbuhay, a web-based Philippine job portal. It provides complete instructions for job seekers, employers, and administrators covering registration, navigation, core features, and troubleshooting. It is also submitted as a semestral output for the GEC IT Laboratory Activity.

1.2 SDG Goal

🌱 UN Sustainable Development Goal 8.6 — Decent Work and Economic Growth

"Substantially reduce the proportion of youth not in employment, education or training (NEET)."

Hanapbuhay directly addresses SDG 8.6 by providing a free digital platform connecting unemployed Filipino youth with employment. By removing financial barriers and providing a built-in resume builder, the system helps reduce the NEET rate especially in underserved communities outside Metro Manila.

1.3 Startup Introduction

Hanapbuhay (Filipino for "to seek a livelihood") is a Philippine-built startup job portal that democratizes access to employment. Unlike commercial job boards with premium fees, Hanapbuhay lets all job seekers browse, apply, and build resumes at zero cost. Employers operate on a sustainable subscription model, keeping the seeker side permanently free.

Built with PHP 8.2, MySQL, and Tailwind CSS, the system is fast, accessible, mobile-friendly, and fully compliant with the Philippine Data Privacy Act (RA 10173).

1.6 Overview of the System

Hanapbuhay is a multi-role web application with three user types: Job Seekers, Employers, and Administrators. It provides a searchable job board, application management, resume builder, subscription-based employer accounts, admin dashboard, and a world-class accessibility widget.

1.7 Scope and Limitations

Scope:
  • Web-based only (no native mobile app in v1.0)
  • Designed for Philippine-based users; three roles: Seeker, Employer, Admin
  • Features: job posting, applications, resume builder, subscription management, notifications
Limitations:
  • No integrated payment gateway — subscriptions activated manually by admin
  • No real-time chat; no AI-based job matching in v1.0
  • Requires active internet (CDN-loaded assets)
  • Maximum file upload: 5MB

4. INSTALLATION GUIDE

⚠️ Before You Begin: Download and install XAMPP 8.2 from https://www.apachefriends.org

Step 1: Start XAMPP Services

1. Open the XAMPP Control Panel.
2. Click Start next to Apache and MySQL. Both must show green status.

Figure 1: XAMPP Control Panel — Apache and MySQL running

Step 2: Copy Project Files

1. Copy the hanapbuhay2 folder into C:\xampp2\htdocs\
2. Final path: C:\xampp2\htdocs\hanapbuhay2\

Figure 2: hanapbuhay2 folder inside htdocs

Step 3: Create the Database

1. Open browser → go to http://localhost/phpmyadmin
2. Click New → type database name: hanapbuhay → click Create
3. Click the Import tab → Choose File → select hanapbuhay2/sql/hanapbuhay.sql
4. Click Import. Wait for the success message.

Figure 3: phpMyAdmin — Importing hanapbuhay.sql

Step 4: Configure Database Connection

1. Open hanapbuhay2/config/database.php
2. Verify: DB_HOST='localhost', DB_NAME='hanapbuhay', DB_USER='root', DB_PASS=''
3. Save the file.

Step 5: Launch the Application

1. Open browser → go to http://localhost/hanapbuhay2
2. The Hanapbuhay homepage will load.
3. Default admin login: admin@hanapbuhay.ph / Admin@123

Figure 4: Hanapbuhay Homepage

7. STEP-BY-STEP PROCEDURES (CORE FEATURES)

7.1 Registration

Purpose: Create a new account as Job Seeker or Employer.

Step 1. Go to http://localhost/hanapbuhay2 → click "Get Started"
Step 2. Select role: Job Seeker or Employer
Step 3. Enter Full Name, Email, Password (min. 8 chars), Confirm Password
Step 4. Click "Create Account"
Step 5. Redirected to your role-specific dashboard automatically

Figure 7: Registration Form

✅ Tip: Employers get a FREE 30-day Professional trial automatically — no credit card needed.

7.2 Login

Purpose: Access your account and personalized dashboard.

Step 1. Click "Sign In" in the navbar or go to /auth/login.php
Step 2. Enter your registered Email and Password
Step 3. Click "Sign In"
Step 4. Redirected to your dashboard (Seeker / Employer / Admin)

Figure 8: Login Page

7.3 Browsing and Searching Jobs

Purpose: Find job listings matching your skills, location, or work type.

Step 1. Click "Find Jobs" in the navbar or use the homepage search bar
Step 2. Enter a keyword (e.g., "Nurse", "Developer") and/or location
Step 3. Use sidebar filters: Job Type, Category, Salary Range
Step 4. Click any job card to view full details
Step 5. Click "Apply Now" or "Save Job"

Figure 9: Job Browse and Search Page

7.4 Applying for a Job

Purpose: Submit an application. Requires a Seeker account.

Step 1. Open a job listing → Review details → Click "Apply Now"
Step 2. Write a Cover Letter
Step 3. Upload resume file (PDF/DOCX) OR use your saved profile resume
Step 4. Click "Submit Application"
Step 5. View confirmation — application appears in My Applications

Figure 10: Job Application Form

7.5 Resume Builder

Purpose: Create a professional resume without external software.

Step 1. Go to Dashboard → Resume Builder
Step 2. Select template: Classic, Modern, or Minimal
Step 3. Fill in: Summary, Work Experience, Education, Skills, Languages, Certifications
Step 4. The right panel shows a live preview updating in real-time as you type
Step 5. Click "Save Resume" to store it; click "Print / Save as PDF" to download

Figure 11: Resume Builder — Live Preview

7.6 Posting a Job (Employer)

Purpose: Create a job listing. Requires an active subscription.

Step 1. Go to Dashboard → Post a Job
Step 2. Fill in: Job Title, Category, Type, Location, Salary Min/Max, Description, Requirements, Deadline
Step 3. Optionally toggle "Mark as Featured"
Step 4. Click "Post Job" — listing appears publicly immediately

Figure 12: Job Posting Form

7.7 Managing Applicants (Employer)

Purpose: Review and update candidate statuses.

Step 1. Go to Manage Jobs → View Applicants
Step 2. See all applicants with name, date, and current status
Step 3. Click an applicant to view their cover letter, resume, and profile
Step 4. Update status via dropdown: Pending → Reviewed → Shortlisted → Hired / Rejected
Step 5. Applicant receives a notification when status changes

Figure 13: Applicant Management

7.8 Admin Dashboard

Purpose: Monitor and manage all platform activity.

Step 1. Login: admin@hanapbuhay.ph / Admin@123
Step 2. Admin Dashboard shows platform statistics (users, jobs, applications)
Step 3. Sidebar navigation: Users, Jobs, Categories, Subscriptions
Step 4. To activate subscription: Subscriptions → find pending record → click Activate

Figure 14: Admin Dashboard

9. FREQUENTLY ASKED QUESTIONS (FAQs)

Q1: Is Hanapbuhay free for job seekers?

A: Yes. All seeker features — browsing, applying, resume builder — are permanently free.

Q2: How do I get my employer subscription activated?

A: New employers get a 30-day FREE Professional trial automatically. After the trial, contact the admin to activate a paid plan.

Q3: Can I use Hanapbuhay on my smartphone?

A: Yes. The site is fully responsive and works on Chrome/Safari mobile. A native app is planned for v2.0.

Q4: How many jobs can an employer post?

A: Basic: 5 jobs | Professional: 15 jobs | Enterprise: Unlimited.

Q5: Can I delete my application after submitting?

A: No — contact the employer or admin to request withdrawal.

Q6: How do I change my password?

A: Go to My Profile → Edit Profile → update password in the security section.

Q7: Who can see my resume?

A: Only employers to whom you have submitted an application. It is not publicly visible.

Q8: Is my data safe on Hanapbuhay?

A: Yes. Compliant with RA 10173 (Data Privacy Act). Passwords use bcrypt encryption.

Figure 15: Login Process Flowchart

Pseudo Code — Login Algorithm

// HANAPBUHAY Login Algorithm
BEGIN LoginProcess
  INPUT email, password
  IF csrf_token invalid → ABORT
  user = QUERY DB WHERE email = input_email
  IF user NOT FOUND → SHOW error → REDIRECT login
  IF bcrypt_verify(password, user.hash) = FALSE → SHOW error
  ELSE
    SET SESSION[user_id] = user.id
    SET SESSION[role] = user.role
    REDIRECT to role_dashboard
END
